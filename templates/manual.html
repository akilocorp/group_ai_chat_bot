ACTR Manual

Group chat for research. Humans join from a survey or the Dashboard. Optional AI personas reply in the room.

Quick start

  1. Open Admin. Click ★ Human-like preset (fills recommended settings), adjust group size and persona names, then Create Session.
  2. Copy the Qualtrics HTML (if Qualtrics is On) or share the session on the Dashboard.
  3. Participants chat. You watch in Monitor and export CSVs from Export.

Typical study room: 1 human + 2 bots → Group size = 1, two personas (a, b), participant name c. Two humans + 2 bots → group size 2, names c,d. Persona names must not match human names.

New sessions use the human-like preset automatically. Old sessions keep their saved settings until you edit them on the Dashboard.

Session modes

Controls which personas may react to each incoming message (human or bot, if Reply on any message is On). The human-like preset uses Session mode 1 (每条消息所有 bot 都可回) and Reply on any message Off.

编号说明（Session vs Persona）： Session Mode 2 = 每条消息只选一个 bot；Session Mode 3 = 仅 @ 时回复。Persona Mode 3 = 随机跳过（与 Session Mode 3 不同）。
  • Mode 1 — all bots: Every persona may reply (human-like preset default). / 每个 bot 都可回复。
  • Mode 2 — one bot per message: The system picks one persona per message. / 每条消息只一个 bot。
  • Mode 3 — @mention only: A persona replies only when its name or @name appears. / 仅 @ 或点名时回复。

A bot never replies to its own message (reduces spam when chains are On).

Bot interaction

Burst protection keeps groups from runaway bot traffic. If the room still feels too “bot-heavy”, lower Max chain depth, raise Bot cooldown, use Session mode 2, or raise persona Mode 3 skip rate. See Natural-looking chat.

Persona modes

Controls when one persona speaks. All modes use delay + typing time.

  • Mode 1 — fixed delay: Always replies after your delay setting.
  • Mode 2 — random delay: Always replies; adds 0.5–2.5 s jitter.
  • Mode 3 — random skip: May skip each message (skip rate). If not skipped, delay + jitter + typing.
  • Mode 4 — context chance: AI estimates reply probability from context, then one random draw (good default for human-like pacing).

Natural-looking chat

If personas talk too much, sound formal, or keep asking questions (e.g. several lines from the same name), click ★ Human-like preset in Admin or match the table in Recommended defaults.

  • Session mode 1 (all bots reply) + Reply on any message On → busiest; avoid for “invisible AI” studies.
  • Persona mode 1 (always reply) → robotic pacing; preset uses Mode 3 (skip) instead.
  • Prompts that say “top 3 ways” → lists and interview questions; use the preset prompt instead.
  • A bot never replies to its own message; that alone does not stop one persona from dominating if others are more eager.

Qualtrics

  1. In Admin, turn Qualtrics On and create a session. Copy the HTML block.
  2. In Survey Flow → Embedded Data, add fields from the table Admin shows (at least session_id and participant_id in the iframe URL; plus condition if Condition is On; plus transcript and chat_status as outputs).
  3. Paste the HTML into a Text/Graphic → HTML question. Use the full inline block (script + iframe), not only an external script tag.
  4. Test with a real anonymous link. Preview may not save transcript.

When data is saved

  • Chat runs full duration → completed_full
  • Participant clicks Next early → left_early or no_messages
  • Close or refresh → same as Next

Transcript text is capped at about 240,000 characters for Qualtrics.

Production server URL is usually https://group.xjhuang.com. Qualtrics cannot load localhost.

Dashboard

  • Sessions — list sessions, join a queue for testing, open settings.
  • Monitor — read-only live view of active rooms (refreshes every few seconds).
  • Export — download activity log or chat history as CSV.

The top metric AI bots is On if any session has bots enabled.

Chat UI shows plain text with a small avatar (👤 or 🤖). There are no chat bubbles.

Modify a published session

Select a session → Edit → change name, timing, session mode, bot interaction caps, or paste updated persona JSON → Save. Bot interaction fields (chain depth, cooldown, etc.) are editable here as well as in Admin at create time. Click ★ Human-like preset in Admin when creating a new session; for an existing one, copy values from Recommended defaults.

FAQ

How do I watch without joining?

Use Dashboard → Monitor.

Can I change a session after creating it?

Use Dashboard → Sessions → open a session → Modify (if the survey window is still open).

Where is data stored?

Local JSON under db/ and activity_logs/, or MongoDB if MONGO_URL is set.

How do I start over?

Create a new session. Old data stays on disk.

Why do bots feel robotic or talk too much?

Usually Session mode 1 plus Persona mode 1 (always reply) and a prompt that asks for “top 3” lists or questions. Apply Recommended defaults or create a new session with ★ Human-like preset.

Do bots know they are talking to other bots?

No special label in the model — everyone appears as a display name in the transcript. Settings treat all senders the same. For a human-led feel, keep Reply on any message Off so only human lines trigger replies.

What do participant_names do?

Optional labels assigned to humans in join order (c, then d, …). Bots use the names you give in personas. Avoid duplicates between human and bot names.
