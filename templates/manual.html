ACTR Manual

ACTR is a group-chat platform for research. Participants join from Qualtrics or the Dashboard. AI personas can reply alongside humans in the same room.

Quick start

  1. Open Admin. Click ★ Example, set Group size and persona names, then Create Session.
  2. After create, copy the Qualtrics HTML block (if Qualtrics is On) from the setup panel below the form.
  3. Participants complete the survey, chat in the embedded room, and click Next. You monitor live traffic on the Dashboard and download CSVs under Export if needed.

Typical room layouts

The Admin form defaults to group size 1 when you click ★ Example.

  • 1 human + 2 bots: Group size = 1, personas a and b, participant name c.
  • 2 humans + 2 bots: Group size = 2, personas a and b, participant names c,d.

Persona names and human names must not overlap (e.g. do not use a as both a bot and a human label).

New sessions start from the ★ Example settings. Existing sessions keep their saved values until you edit them on the Dashboard.

Session modes

Session mode decides which personas are allowed to respond to each incoming message (from a human or, if Reply on any message is On, from another bot).

Do not confuse with persona mode. Session mode 3 = @mention only. Persona mode 3 = random skip — a different setting on each persona card.
  • Mode 1 — all bots (preset): Every persona may be queued to reply to each eligible message. Combined with persona mode 3 (skip), not every bot will actually speak every time.
  • Mode 2 — one bot per message: The system picks a single persona per message. Calmer than mode 1.
  • Mode 3 — @mention only: A persona replies only if its name or @name appears in the message text.

A persona never replies to its own message (reduces self-triggered chains).

Group chat flow

Session-level controls (Reply on any message, Max chain depth) sit in the AI Personas section on Admin. Per-persona Delay spaces replies; there is no separate bot cooldown or per-minute cap.

How a message flows

  1. Someone sends a line over WebSocket → saved to history and context.
  2. Session mode decides which personas are eligible (all, one, or @mentioned only).
  3. Each eligible persona runs its persona mode (delay, skip, or probability).
  4. If Reply on any message is On and max chain depth allows, bot replies can trigger further bot rounds until the depth limit is reached.

If the room feels noisy, lower chain depth, increase each persona’s delay, raise skip rate, or switch to session mode 2.

Persona modes

Persona mode applies per persona after session mode has already decided eligibility.

  • Mode 1 — fixed delay: Always replies (after delay + typing) when eligible.
  • Mode 2 — random delay: Always replies; adds 0.5–2.5 s jitter before generation.
  • Mode 3 — random skip (preset): Each eligible message: roll against skip rate; if not skipped, reply with delay + jitter + typing.
  • Mode 4 — context chance: Model estimates reply probability from context, then one random draw. Good when you want fewer but more “thoughtful” replies.

Qualtrics

Setup steps

  1. In Admin, set Qualtrics to On and create a session.
  2. In Survey Flow → Embedded Data, create the In and Out fields listed below (at minimum).
  3. Paste the generated HTML (script + iframe) into a Text/Graphic → HTML question.
  4. Publish and test with an anonymous link. Survey Preview often does not save transcript correctly.

Fields you must create in Qualtrics

FieldDirectionNotes
session_idIn (iframe URL)Fixed in the HTML snippet from Admin (e.g. SES-…).
participant_idIn (iframe URL)Use ${e://Field/ResponseID} or your participant key.
conditionIn (iframe URL)Required when Condition is On in Admin. Same value → same match queue.
transcriptOut (saved)Written by the chat app when the session ends (~240k char cap).
chat_statusOut (saved)completed_full, left_early, no_messages, or never_joined.

Only session_id, participant_id, and (if used) condition are passed in the iframe URL. transcript and chat_status are written back by the app. Server-only settings (group size, session mode, personas, etc.) are documented in the Admin setup table after create — they are not sent through Qualtrics automatically.

When data is saved

  • Participant stays until chat duration ends → completed_full
  • Clicks Next before the room ends → left_early or no_messages
  • Closes tab or refreshes → same as Next (handoff script runs on unload when configured)
Production embed URL is typically https://group.xjhuang.com. Qualtrics cannot load localhost.

Dashboard

  • Sessions — join a queue for testing, open published session settings.
  • Monitor — read-only live view of active rooms.
  • Export — download activity logs or chat history as CSV (optional; Qualtrics Out fields are the primary handoff when integration is On).

Modify a published session

Select a session → Edit → change timing, session mode, bot interaction limits, or persona JSON → Save. Use Admin ★ Example when creating a new session; for existing sessions, align fields with Recommended defaults.

FAQ

How do I watch without joining?

Dashboard → Monitor → select a room.

Can I change settings after create?

Yes, while the survey window is open: Dashboard → Sessions → your session → Edit.

Where is data stored?

Local JSON under db/ and activity_logs/, or MongoDB if MONGO_URL is set.

Why do bots feel robotic or talk too much?

Apply ★ Example in Admin or see Recommended defaults.

Do bots know they are talking to other bots?

No special label is shown to the model — everyone is a display name in the transcript. For human-led flow, keep Reply on any message Off.

What is the difference between session mode and persona mode?

Session mode = who may reply to a message. Persona mode = whether this persona actually replies when eligible.
